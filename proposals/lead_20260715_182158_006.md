**[v2][macOS] Sign and notarize Airo TV for public consumer release**

*Project description:*
## Summary

The Airo TV v0.0.3 macOS DMG release asset is downloadable and internally code-signature-valid, but the public new-consumer journey fails at Gatekeeper because the app is unsigned/not notarized.

This issue tracks the human-required Developer ID setup and the next-release automation run needed to publish a notarized macOS artifact.

## Current Evidence

Release tested: `airo-tv-v0.0.3`

Public asset tested:

`https://github.com/DevelopersCoffee/airo/releases/download/airo-tv-v0.0.3/Airo-TV-0.0.3-macOS.dmg`

Observed results:

- GitHub release page renders and exposes the macOS DMG asset.
- Direct DMG download succeeds.
- DMG checksum matches the release digest:
  `e9a5467f656a6e85f27328ca2770039442bc2602216ead2db53c90d2f141e036`
- DMG mounts cleanly.
- `Airo TV.app` copies out of the DMG.
- `codesign --verify --deep --strict --verbose=2` reports the copied app is valid on disk.
- `spctl --assess --type execute --verbose=4` rejects the copied app.
- First normal `open` attempt from a quarantined browser download does not start the app.
- After `xattr -dr com.apple.quarantine`, the app launches successfully.

Conclusion: the app binary is runnable, but the public consumer macOS journey is blocked until Developer ID signing and notarization are in place.
