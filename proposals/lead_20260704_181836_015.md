*Project description:*
## What happened

Two `test/vitest/vitest.infra.config.ts` tests fail deterministically on a pristine macOS checkout of latest `main` while Linux CI is green on the same SHA. This breaks the `scripts/pr prepare-run` local `pnpm test` gate for every PR landed from a Mac.

Repro (macOS, clean clone of `main`, sanitized env, Node 24 or 25):

```
node scripts/run-vitest.mjs run --config test/vitest/vitest.infra.config.ts src/infra/exec-approvals-analysis.test.ts src/infra/backup-create.test.ts
```

1. `src/infra/exec-approvals-analysis.test.ts` > "Windows enforced shell command rendering > builds enforced command for simple Windows command": the test resolves the host `python3`. On macOS/Homebrew that is `/opt/homebrew/Cellar/python@3.14/.../bin/python3.14` — the `@` makes the renderer (correctly) quote the executable, and the assertion regex `/^& .+python3(?:\.\d+)? a\.py$/` does not allow the closing quote before ` a.py`.

2. `src/infra/backup-create.test.ts` > "fails when SQLite-looking state appears after snapshot discovery: late.sqlite(-wal)": the `fs.readdir` spy injects the late file only when the scanned dir equals `path.resolve(state.stateDir)`. Snapshot discovery scans the realpathed state dir; on macOS the tmpdir-based `stateDir` goes through the `/var -> /private/var` symlink, so the comparison never matches, the late file is never injected, and the archive resolves instead of rejecting.

Both are test-environment bugs, not product bugs: the quoting behavior is correct, and the appears-after-snapshot guard works when it actually sees the late file.

## Impact

`pnpm test` cannot go green on a clean macOS checkout of `main`, so `scripts/pr prepare-run`/`merge-run` cannot produce gate stamps on maintainer Macs. Found while landing #99935.
