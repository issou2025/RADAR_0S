# Project Proposal — Flutter App

## Project Understanding
You need assistance with: 
**feat(web/flutter): add Google and GitHub OAuth sign-in**

*Project description:*
## Overview
Login and signup support only email + password. Adding OAuth providers (Google and GitHub) reduces friction for new users. Supabase supports both with minimal backend setup.

## What needs to be done

### Supabase config
- [ ] Enable Google OAuth in Supabase dashboard: Auth → Providers → Google — add Client ID + Secret from Google Cloud Console
- [ ] Enable GitHub OAuth: Auth → Providers → GitHub — add Client ID + Secret from GitHub Developer Settings
- [ ] Add redirect URL `https://echomirrorbutler.vercel.app/auth/callback` to both providers

### Auth callback route (web)
- [ ] Add `/auth/callback` route in `router.tsx` (public, no auth guard)
- [ ] Callback component calls `supabase.auth.exchangeCodeForSession()` on mount
- [ ] On success: redirect to `/dashboard` (or `/onboarding` if `onboarding_completed` is not set)
- [ ] On error: redirect to `/login?error=oauth_failed`

### Login page
- [ ] Add "Continue with Google" and "Continue with GitHub" buttons above the email form
- [ ] Use official SVG logos (not emoji or font icons)
- [ ] Add an "or" divider between OAuth section and email form
- [ ] Call `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: '/auth/callback' } })`

### Signup page
- [ ] Same OAuth buttons at the top
- [ ] New OAuth users (no `onboarding_completed`) are routed to onboarding

### Flutter
- [ ] Add `google_sign_in: ^6.x` to `pubspec.yaml`
- [ ] Implement Google sign-in on the Flutter login screen using Supabase's `signInWithIdToken`
- [ ] Add SHA-1 fingerprint to Google Cloud Console for Android

## Acceptance criteria
- Users can sign up and log in with Google and GitHub on web
- New OAuth users are routed to onboarding
- Returning OAuth users land on dashboard
- `/auth/callback` handles errors gracefully
- Flutter supports Google sign-in
