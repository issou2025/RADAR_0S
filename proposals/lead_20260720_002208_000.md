## What to build

Completely decouple `MergeAnalysisEngine.cs` from the active Revit database and Revit API types, allowing duplicate clustering and naming analysis to run headlessly on pure `ElementModel` POCOs.

## Acceptance criteria

- [ ] Move database queries (such as scanning for family instances, locations, and bounding boxes) out of the pure matching logic and encapsulate them inside a Revit-connected data collector helper.

- [ ] Refactor bounding box size comparisons and centroid offset calculations to perform vector arithmetic directly on `XYZModel` and `BoundingBoxXYZModel` POCOs instead of calling `.ToNative()!`.
- [ ] Ensure safe null dereferencing guards on all POCO property operations inside the matching loops.
- [ ] Replace any single-letter variables in `MergeAnalysisEngine.cs` with descriptive names.
- [ ] Resolve data clump smells by bundling `paramGroup` and `paramType` into a cohesive type.
