*Project description:*
## Context

This is a consolidated wishlist from the perspective of a developer evaluating
and integrating the V-Modal Flutter SDK. The main goal is to make the path from
"What does V-Modal do?" to a secure, production-ready video search integration
clear and reproducible.

## Documentation requested

### 1. Product and architecture overview

- Explain what V-Modal does in practical terms and the main use cases.
- Show the full data flow: authenticate -> upload -> index -> discover the
  searchable version -> search -> display/open a result.
- Clarify what runs on the device, in the SDK, and in the V-Modal backend.
- Include a glossary for `collection`/`group`, `stream`/`sub_collection`,
  `mode`, `modality`, `index_type`, and `version_lancedb`.

### 2. Installation and compatibility

- A minimal Flutter installation guide that works in a fresh public clone.
- Supported Flutter/Dart versions and Android/iOS minimum versions.
- Android Studio setup, including the Flutter and Dart plugins.
- A version compatibility table for the SDK, Flutter, Dart, Android Gradle
  Plugin, Gradle, Kotlin, and iOS tooling where applicable.

### 3. Authentication and security

- Explain how API keys are issued, scoped, rotated, and revoked.
- Show the recommended production pattern for supplying a key without
  committing or embedding a long-lived secret in the application.
- Document test/staging/production endpoints and how to configure them.
- State data retention, deletion behavior, supported regions, and relevant
  privacy/security expectations.

### 4. End-to-end lifecycle reference

- Document the required order and state transitions for upload, indexing, and
  search.
- Explain when an upload is complete versus when an index is searchable.
- Define all index job states, terminal states, polling recommendations, and
  retry behavior.
- Explain how to select `version_lancedb` and when versions change.
- Document collection/stream naming rules and whether names must already exist.

### 5. Complete API and model reference

- One page per public SDK client/resource/method with parameters, defaults,
  validation rules, return types, and examples.
- A response-field reference for search results, especially `score`,
  `score_ui`, `ts_unix`, playback/frame offsets, `image_url`, `thumbnail_url`,
  and `sample_images`.
- Explain whether scores are similarity, distance, rank normalization, or
  confidence, including whether higher or lower is better.
- Document pagination, ordering, filters, supported search sources, and combine
  modes.
- List supported upload formats, codecs, sizes, durations, rate limits, quotas,
  timeouts, and expected processing time.

### 6. Errors, logging, and troubleshooting

- A status-code/error-code table with causes and recommended user actions.
- Troubleshooting guides for authentication failures, missing collections,
  missing index versions, search 404s, stalled jobs, and upload failures.
- Explain how to enable SDK logs in Flutter, Android Logcat, and Dart/Flutter
  DevTools without exposing API keys or signed URLs.
- Include request/correlation IDs in examples so backend problems can be
  reported effectively.

### 7. Releases and maintenance

- Changelog and migration guides for breaking changes.
- Deprecation and semantic-versioning policy.
- Published roadmap/support policy and a clear way to report SDK versus backend
  issues.

## SDK and example features requested

- Typed search-result models instead of requiring consumers to read raw maps.
- A result widget/example that renders the matched thumbnail/frame and can open
  the video at the matching moment. Related: #11.
- A high-level upload -> index -> wait-until-ready helper, while retaining the
  lower-level APIs.
- A job-status stream or configurable polling helper with timeout and
  cancellation.
- Structured, opt-in logging callbacks with automatic redaction of API keys,
  authorization headers, and signed URLs.
- Configurable retry/backoff and clear transient versus permanent errors.
- Progress reporting for upload and indexing, including background/lifecycle
  behavior.
- First-class pagination and filters for larger result sets.
- Methods to list, inspect, and safely delete uploaded assets and indexes.
- Dependency-injection/mocking interfaces and fixtures for application tests.
- A production-quality example UI with loading/empty/error states,
  accessibility, safe-area handling, and result previews.

## Tutorials requested

1. **Ten-minute Flutter quickstart** — configure securely, upload the bundled
   sample, create an index, search it, and render a result.
2. **Search an existing collection** — skip upload and correctly discover the
   collection, stream, and latest searchable version.
3. **Use a real video** — supported formats, file selection, upload progress,
   indexing time, cancellation, and cleanup.
4. **Build a video-search results screen** — thumbnails, scores, timestamps,
   pagination, empty states, and playback at the matched moment.
5. **Production authentication** — obtain short-lived credentials through an
   application backend and rotate/revoke them safely.
6. **Debug common failures** — practical examples for 401/403/404/429/5xx,
   Logcat filters, DevTools, and redacted request logging.
7. **Test an integration** — mocks for unit/widget tests plus an opt-in live
   integration test that creates and cleans up temporary resources.
8. **Performance and reliability** — retry policy, timeouts, polling intervals,
   caching, large videos, backgrounding, and mobile network interruptions.
9. **Image, text, OCR, and ASR search modes** — what each source searches and
   how union/combined ranking behaves.
10. **Migration tutorial** — upgrading SDK versions without changing production
    behavior unexpectedly.
