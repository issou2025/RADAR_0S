## Phase 2: Embedded inference (free tier, zero-install)
- [ ] Prototype LLamaSharp in out-of-process host (NOT inside Revit first — isolate native dll risks)
- [ ] Evaluate ONNX Runtime GenAI + DirectML as alternative (no-NVIDIA machines)
- [ ] Model download manager: GGUF fetch on first use, checksum, progress UI
- [ ] Hardware probe: RAM/GPU detection → recommend model size or disable embedded tier
- [ ] Benchmark on realistic office hardware: prefill + generation tok/s, Qwen3-4B vs Gemma small
- [ ] Decision gate: is embedded chat viable, or embedded = embeddings-only?

## Phase 3: RAG — ingestion pipeline (runs outside Revit)
- [ ] Choose stores: sqlite-vec (local/free) + pgvector on Supabase (paid/collab)
- [ ] VectorStore abstraction over both (upsert, query, filter by metadata)
- [ ] Embedding model: bge-m3 or multilingual-e5 (DE/RU/EN), store model name in DB metadata
- [ ] PDF text extraction + chunking for meeting minutes (chunk per agenda item / TOP)
- [ ] Metadata schema: source file, page, date, project, discipline, doc type
- [ ] Ingestion worker (console app or Aspire service): watch project folder, index new/changed files
- [ ] Re-index command (full rebuild on embedding model change)
- [ ] Later sources: CONTACT Elements docs (reuse ContactApiService), Excel specs, Raumbuch

## Phase 4: RAG — retrieval & search UX
- [ ] Hybrid search: FTS5/BM25 + metadata filters → vector search top-30
- [ ] Cross-encoder reranker (bge-reranker) → top 3-5 chunks
- [ ] Search-only mode (no LLM): semantic search results with source links — fast on any hardware
- [ ] "Summarize with AI" action on top of search results (optional LLM step)
- [ ] Answer view with citations: doc name + page, click → open source file
- [ ] Token budget guard: hard cap on context size per request
