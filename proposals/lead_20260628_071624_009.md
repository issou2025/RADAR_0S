## Motivation

Forward-looking tracking issue for scraper anti-bot resilience. **Not for immediate implementation** — captured so the analysis isn't lost. ## Current state (baseline)

The scraper (`scraper/main.py`) launches a **single global Chromium at lifespan** (`:438-457`) and creates `new_context()` per request (`:712`). Existing, modest, hand-rolled stealth:
- `--disable-blink-features=AutomationControlled` native launch flag (`:445-455`) — correctly chosen over `add_init_script` prototype-patching, which CF can detect.
- Pinned `_BROWSER_USER_AGENT` (`:589-591`) + realistic viewport/locale/Accept-Language (`:711-713`).
- Cloudflare managed-challenge detection via `window._cf_chl_opt` (`:625-627`).

Not present: any stealth/fingerprint library, proxy rotation, browser recycling.

## Gaps identified (all language-independent — no Node/TS migration needed; every fix below is first-class Python)

1. **Shared fingerprint + TLS (JA3/JA4) across contexts.** One Chromium binary ⇒ every context shares the same hardware fingerprint *and* TLS cipher ordering. Latent today (we don't rotate proxies, so we present as one consistent client); becomes the "many users / one fingerprint" tell once proxy/cookie rotation + high concurrency arrive at Phase 4. Note: switching host language would NOT change JA3 — same binary = same TLS signature; mitigation is a fingerprint library and/or TLS-impersonating proxy and/or hardened fork, not a rewrite.
2. **No browser recycling.** The lifespan browser runs indefinitely → memory fragmentation + a continuously-mappable browser lifetime. *(This is the one item worth doing sooner — it's memory hygiene independent of anti-bot; see below.)*
3. **Pinned-UA drift.** `_BROWSER_USER_AGENT` will drift from real Chrome over time (already noted in-code).

## Candidate mitigations (evaluate when triggered, cheapest first)

- **Per-context fingerprint randomization** — e.g. `browserforge` (statistically realistic fingerprints injected per `new_context()`); also addresses UA drift.
- **Browser recycling** — recycle (close + relaunch) the global browser after N requests / a time budget.
- **Hardened browser fork** — evaluate **Patchright** (patched Playwright, strips automation tells at the binary level, drop-in for Python) or **Camoufox** (hardened Firefox) only if standard Chromium + fingerprint randomization proves insufficient.
- **Proxy rotation** — residential/rotating proxies; pair with per-session fingerprint so rotated IPs don't all share one hardware hash.

## Explicitly out of scope / discount

- **Do not** add stealth machinery speculatively (reproduce-before-blaming: each layer is its own maintenance + detection surface). Add only against measured block rates.
- This is **not** a reason to migrate the scraper to Node/TS — all mitigations are Python-available, and the JA3/fingerprint vectors are binary-level, not language-level. (See the keep-Python decision.)

## Suggested near-term carve-out

**Browser recycling** (gap #2) is the only piece worth doing independent of anti-bot — it's operational memory hygiene for the long-lived global browser. Consider splitting it into its own small issue if/when scraper memory becomes a concern.
