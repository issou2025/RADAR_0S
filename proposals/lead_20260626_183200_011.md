### Issue #1 (Deep Dive) — "Consensus Divergence Index: Measuring Signal in Multi-Model LLM Swarms (8k)"
This is the **theoretical bedrock** of the whole project. Key concepts:

1. **CDI (Consensus Divergence Index):** A mathematical ratio of disputed claims to total claims. High disagreement between LLM architectures (Claude, Gemini, GPT, etc.) = real research boundary.
2. **Consensus Trap:** When models agree, it's often sycophancy or shared training data mirroring — **not** truth.
3. **Leapfrog Mechanism:** Externalizing state to GitHub Issues as atomic, stateless reentry points — reducing context re-entry time from ~23 minutes to <3 minutes.
4. **Recursive Agent Consensus (RAC):** Using architectural diversity as an adversarial audit layer.
5. **Hardware Independence:** High-rigor synthesis on consumer hardware (16GB RAM).

The issue has rich multi-model commentary from **Gemini**, **Qwen3 Coder**, and **Claude** — and Claude caught something brilliant: the Qwen3 response was a near-verbatim copy of Gemini's, which is *literally the Consensus Trap happening inside the issue about the Consensus Trap*. CDI = 0 between them. 🧠

Claude also did a rigorous triage:
- The "23→3 min" claim needs honest framing: **"self-reported by single practitioner"** — 0 actual measured datapoints
- Real stats: **311 total issues, 238 hours, ~1.16 issues/hour over 9 days**
- Claude offered to write a Python script to actually measure re-entry times (SQLite, two `input()` prompts, a SQLite table)
