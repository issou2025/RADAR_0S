# Project Proposal — Bim Modeling

## Project Understanding
You need assistance with: 
**Family Manager — dockable placement palette (second variant)**

*Project description:*
Goal: besides the current standalone Family Manager window, add a DOCKABLE pane
whose purpose is PLACING families, not auditing. It shows types grouped by family
with previews; clicking a type starts Revit's placement flow. It is a different
scenario from the window, not just a smaller size.

### Two distinct surfaces (do NOT merge into one page)
- [ ] Window = audit/cleanup (rename, delete, purge, rules, 3D). Keep as-is.
- [ ] Dock  = placement palette (types grouped by family, search, previews, "place").
      Do NOT drag rules / purge / 3D viewer into the dock.

### Dockable pane plumbing
- [ ] Register a `DockablePane` (IDockablePaneProvider) with a stable GUID at add-in
      startup (OnStartup — cannot be registered lazily on button click).
- [ ] Host the clientapp at a NEW route, e.g. `#/families-dock`, in the pane's WebView2.
- [ ] Reuse the same WebView2 transport + dispatcher as the window so AT.invoke works.
- [ ] Reuse the shared WebView2 profile/origin → shared preview cache (IndexedDB) and
      AI-model store (localStorage).
- [ ] One pane instance is shared across documents; refresh it on active-doc change.

### New palette UI (thin, placement-focused)
- [ ] New component `FamilyPaletteView`: tree/accordion "Family → its types",
      search box, preview grid.
- [ ] Primary action per type = Place. Secondary maybe: select/isolate existing instances.
- [ ] Optional: "recent" / "favorite" families for quick access.

### New backend command
- [ ] `PlaceFamilyInstance(typeId)` → resolve FamilySymbol, activate it if needed,
      then `uidoc.PromptForFamilyInstancePlacement(symbol)`.
- [ ] Loadable families only. System types (walls/floors/pipes) are drawn, not placed
      from a palette → exclude them from the palette (or a separate branch later).

### Reuse (data + small components, NOT layout)
- [ ] Reuse `GetFamilyTypeRows` (already returns type + family + preview keys).
- [ ] Reuse `FamilyThumb` for previews.
- [ ] Reuse shared preview cache + AI-model store via the common WebView2 profile.

### Ribbon / UX
- [ ] Add a way to open/show the dockable pane (ribbon toggle, or a split button on the
      existing Family Manager button: "Window" vs "Docked").
- [ ] Verify WebView2 inside the docked host: sizing, DPI, focus.
