# Project Proposal — Bim Modeling

## Project Understanding
You need assistance with: 
**Door flip (Shift+F) switches the opening side; add a way to flip the hand while keeping the opening direction**

*Project description:*
### Summary

When flipping a door with Shift+F, Bonsai flips the door to the other face of the wall, which also changes the opening direction. There are really two distinct flips a user needs, and only one is available today.

### The two flips
1. **Flip the hand (hinge side).** The door stays on the same face and keeps opening the same direction, but the hinge moves to the other jamb. The swing mirrors left to right.
2. **Flip the facing (side).** The door moves to the other face of the wall and opens the other way, into the room versus out.

Bonsai currently does the facing flip. There is no way to flip only the hand while keeping the opening direction, which is the more common everyday operation for most users.

### Reference

Revit and ArchiCAD both expose these as separate controls, flip hand and flip facing, because architects genuinely need both.

### Proposal
- Provide both flips: flip the hand (keep the opening direction, mirror the hinge) and flip the facing (switch sides), as separate options.
- Keep the IFC OperationType consistent with the result, for example SINGLE_SWING_LEFT versus SINGLE_SWING_RIGHT for a hand flip, so the 2D symbol and the schema stay in sync.

### Context

This came up while looking at #4950. The current single flip is easy to misread because it changes the opening direction rather than just the hand. Raising it here as a design question: how would you like Shift+F and the flip controls to behave? Happy to help implement whichever direction you prefer.
