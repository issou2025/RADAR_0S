# Project Proposal — Flutter App

## Project Understanding
You need assistance with: 
**Implement MPIN Setup API Integration**

*Project description:*

Integrate the **MPIN Setup** API. This is the final step of the mobile app onboarding. It securely hashes a 4-digit PIN for the user. Once this succeeds, the user is given full access and refresh tokens, meaning they are now fully logged into the app.

## Endpoint Details
*   **Method:** `POST`
*   **URL:** `{{BASE_URL}}/api/v2/mobile/auth/mpin-setup`
*   **Headers:**
    *   `Content-Type: application/json`
    *   `Authorization: Bearer <TEMP_TOKEN>` (Retrieved from the Verify OTP API response)

## Request Body
```json
{
  "mpin": "1234",
  "confirmMpin": "1234"
}
```

---

## Expected Responses & Status Codes

### 1. Success (201 Created)
MPIN is saved, and the user is officially logged in!
**Action required by Flutter Dev:** 
- Clear the `tempToken` from secure storage (it's no longer needed).
- Save the `accessToken` and `refreshToken` in secure storage. 
- Save the `customer` details (like name and ID) in global state/Hive/SharedPreferences.
- Navigate the user to the **Main App Dashboard**.

```json
{
    "success": true,
    "message": "MPIN set successfully. You are now logged in.",
    "data": {
        "accessToken": "eyJhbGciOi...",
        "refreshToken": "a8b7c6d5...",
        "customer": {
            "id": "uuid-xxx-xxx",
            "customerId": "CUS123456",
            "fullName": "John Doe",
            "phoneNumber": "9876543210"
        }
    },
    "error": {}
}
```

### 2. Validation Error (400 Bad Request)
If the PINs don't match, aren't 4 digits, or are missing.
**Action required by Flutter Dev:** Show a local UI error. (Note: You should also validate this on the frontend *before* making the API call to save time).
```json
{
    "success": false,
    "message": "MPIN and Confirm MPIN do not match",
    "data": {},
    "error": "MPIN mismatch"
}
```

### 3. Unauthorized (401 Unauthorized)
If the `tempToken` is missing or expired.
**Action required by Flutter Dev:** Show a SnackBar saying "Session expired", clear local state, and send the user back to the Phone Number entry screen.
```json
{
    "success": false,
    "message": "Temporary authentication failed. Please verify OTP again.",
    "data": {},
    "error": "jwt expired"
}
```
