## Description

## Mathematical & Design References

All signal processing math is implemented in Part 1 (DSP Core Library). This part integrates it into a Flutter app.

| Concept | DSP Core Function | Flutter Integration |
|---------|------------------|-------------------|
| Nyquist Sampling Theorem (f_s ≥ 2·f_max, so 44.1 kHz ≥ 2·22 kHz) | miniaudio config | AudioService sets sampleRate=44100 |
| FFT Spectrum (N-point DFT for visualization) | sw_get_spectrum() | SpectrumAnalyzer widget renders FFT bins |
| Constellation (IQ scatter plot of equalized symbols) | OFDM demodulator output | ConstellationDiagram widget plots IQ points |
| CFO Estimation (Δf̂_c = f_s/(2πN)·∠(Σ y*[n]·y[n+N])) | sync_schmidl_cox_cfo() | Display estimated CFO in receiver UI |
| SNR Estimation (10·log₁₀(P_s/P_n)) | diag_estimate_snr() | SNR display in connection status widget |
| BER Estimation | diag_estimate_ber() | BER display in connection status widget |
| Matched Filter Detection (R_ys[m] = Σ y[n+m]·s*[n]) | sync_detect_preamble() | Connection state: LISTENING → SYNCED |
| Threshold Detection (|R_ys|² > θ·σ²_n) | sync_detect_preamble() | Connection state: SYNCED → RECEIVING |
| CRC-32 Verification (G(x) polynomial) | crc32_verify() | TransferResult.success indicator |
| RS Error Correction (t = ⌊(n-k)/2⌋) | rs_decode() | Error-resilient data recovery |

### 2.2 SoundwaveBridge Wrapper Class

- [ ] Create `SoundwaveBridge` singleton class:
```dart
class SoundwaveBridge {
  static final SoundwaveBridge _instance = SoundwaveBridge._();
  factory SoundwaveBridge() => _instance;
  
  SoundwaveBindings? _bindings;
  Pointer<SwConfig>? _configPtr;
  Pointer<Void>? _contextPtr;
  DynamicLibrary? _lib;
}
```

- [ ] Implement `void _initializeBindings()`:
  - Load native library per platform
  - Initialize `SoundwaveBindings(_lib)`
- [ ] Implement `void createContext(PhyConfig config)`:
  - Allocate `Pointer<SwConfig>` via `malloc.sizeOf<SwConfig>()`
  - Set fields: carrierFreq, bandwidth, sampleRate, symbolRate, modulation, ofdmSubcarriers, cpLength, rsN, rsK
  - Call `_bindings.sw_create(_configPtr)` → store `_contextPtr`
  - Handle null return (error during creation)
- [ ] Implement `void destroyContext()`:
  - Call `_bindings.sw_destroy(_contextPtr)`
  - Free config: `malloc.free(_configPtr!)`
- [ ] Implement `Uint8List encode(Uint8List data)`:
  - `Pointer<Uint8> dataPtr = allocate(data)` (helper)
  - `Pointer<SwBuffer> result = _bindings.sw_encode(_contextPtr, dataPtr, data.length)`
  - Convert SwBuffer to Uint8List: `result.ref.data.asTypedList(result.ref.length)`
  - Free native buffer: `_bindings.sw_free_buffer(result)`
  - Return Dart copy of data
- [ ] Implement `Uint8List decode(Uint8List encoded)` — same pattern
- [ ] Implement `Float32List modulateCss(Uint8List bits)`:
  - Convert bits to `Pointer<SwBuffer>`
  - Call `sw_modulate_css` → returns `Pointer<SwSignal>`
  - Extract float array: `result.ref.data.asTypedList(result.ref.length)`
  - Return Float32List copy
- [ ] Implement `Float32List modulateOfdm(Uint8List bits)` — same pattern
- [ ] Implement `Uint8List demodulate(Float32List samples)`:
  - Pass float array pointer to `sw_demodulate`
  - Return decoded Uint8List
- [ ] Implement `void startAudio()` / `void stopAudio()`:
  - Call `sw_audio_start` / `sw_audio_stop`
- [ ] Implement `double getSnr()` / `double getBer()`:
  - Call `sw_get_snr` / `sw_get_ber` → return float
- [ ] Implement `SignalData getSpectrum()`:
  - Call `sw_get_spectrum` → extract magnitudes, minFreq, maxFreq, numBins
  - Return `SignalData(frequencyDomain: magnitudes, ...)`

## 7. CSS End-to-End Integration (Desktop)

### 7.1 TX Flow

- [ ] User enters text → `PhyConfig(carrierFreq, bandwidth, modulation=CSS)`
- [ ] `SoundwaveBridge.createContext(config)`
- [ ] `SoundwaveBridge.encode(textUtf8)` → encoded bytes with RS + CRC
- [ ] `SoundwaveBridge.modulateCss(encodedBits)` → Float32List of chirp samples
- [ ] `SoundwaveBridge.startAudio()` → begins playback
- [ ] Write chirp samples to ring buffer → played through speaker
- [ ] After TX completion: `SoundwaveBridge.stopAudio()`

### 7.2 RX Flow

- [ ] `SoundwaveBridge.createContext(config)`
- [ ] `SoundwaveBridge.startAudio()` → begins capture
- [ ] Microphone captures → samples written to ring buffer
- [ ] Periodically (every 1024 samples): check for preamble via `sync_detect_preamble()`
- [ ] Preamble detected → connection state = SYNCED
- [ ] Continue capturing → `SoundwaveBridge.demodulate(capturedSamples)` → decoded bits
- [ ] `SoundwaveBridge.decode(decodedBits)` → recover payload
- [ ] CRC verify → TRANSFER_RESULT with success/fail

### 7.3 Testing

- [ ] Test: CSS TX between two desktop instances on same machine (loopback via audio cable or virtual audio device)
- [ ] Test: CSS TX between two physically separate machines
- [ ] Test: CSS with ambient noise (fan, conversation)
- [ ] Measure: CSS throughput and BER

## 9. Binary Payload Support

### 9.1 Fragmentation

- [ ] Max payload per frame: 4 KB (4096 bytes)
- [ ] Payloads > 4 KB divided into 4 KB chunks
- [ ] Header prepended to each chunk: `[uint16 sequence_num, uint16 total_chunks, uint16 chunk_size]`
- [ ] Chunks transmitted sequentially

### 9.2 Reassembly

- [ ] Receive buffer: accumulates chunks by sequence number
- [ ] On receiving all chunks: reassemble in order
- [ ] CRC-32 verify on complete payload
- [ ] Timeout: if all chunks not received within 60s → declare failure

### 9.3 File Picker

- [ ] `file_picker` package: `FilePicker.platform.pickFiles(type: FileType.any)`
- [ ] Limit file size to 1 MB
- [ ] Show file name and size before TX

### 9.4 Transfer Progress

- [ ] Progress bar: chunk N of M
- [ ] Estimated time remaining
- [ ] Throughput in bps

### 9.5 Retransmission

- [ ] On CRC failure for a chunk: retransmit up to 3 times
