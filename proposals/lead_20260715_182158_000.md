### 🐞 Describe the bug

## Issue Description

The `pyrevit caches bim360 clear` command does not respect custom cache locations configured in Revit.ini for Revit 2024+. The Python library (`bim360.py`) has support for reading custom `CacheLocation` from the `[CloudModelCache]` section in Revit.ini, but the CLI command implementation in C# does not utilize this functionality. This means users with custom BIM360 cache locations cannot properly clear their caches via the CLI.

**Expected Behavior:**
The CLI should read the custom cache location from Revit.ini (if present) for Revit 2024+ and clear caches from that location, falling back to the default `%LOCALAPPDATA%/Autodesk/Revit/Autodesk Revit {version}/CollaborationCache` path if not configured.

## Implementation Suggestion

The implementation would need to:

1. Add a method in the C# codebase to read Revit.ini and extract the `CacheLocation` value from the `[CloudModelCache]` section (similar to the Python `_get_custom_cache_root` function) [4](#0-3) 
2. Modify `RevitCaches.ClearCache()` and `RevitCaches.ClearAllCaches()` to check for custom cache locations for Revit 2024+
3. Update the CLI autocomplete patterns if needed [5](#0-4) 

The key constants needed are:
- Revit.ini path format: `%APPDATA%/Autodesk/Revit/Autodesk Revit {version}/Revit.ini` [6](#0-5) 
- INI section: `CloudModelCache` [7](#0-6) 
- INI key: `CacheLocation` [8](#0-7) 
- Minimum version: 2024 [3](#0-2) 

## Notes

The Python implementation already has the logic for custom cache locations [9](#0-8) , so the C# implementation would need to mirror this functionality. The current CLI command pattern does not need to change since the custom cache location is determined automatically from Revit.ini, not via command-line arguments.
