Hi, 

I found this project while looking through Procore automation repos. I maintain PyProcore, an open-source Python SDK for Procore:
https://github.com/vibhanshu-mishra/pyprocore

It handles a lot of the plumbing this backend may eventually need:
- OAuth authorization-code flow
- token refresh
- retries
- pagination
- typed response models
- project/RFI/submittal/document/drawing/photo/daily-log helpers
- CLI diagnostics
- local exports and workflow packages

Since this repo is already using FastAPI + PostgreSQL + SQLAlchemy + Procore API + OAuth, PyProcore might be useful as the Procore client layer instead of maintaining custom request/auth logic.

I’d be happy to open a small PR if helpful, maybe something like:

- update `/projects`
- update `/projects/{id}/submittals`
- add a README note showing the SDK option
