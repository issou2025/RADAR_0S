## Feature Request: Add Baidu Comate (文心快码) as a PROVIDER (Baidu's free coding assistant — extract from VS Code extension)

### Summary

Add **Baidu Comate (文心快码)** (`BaiduComate.comate`) as a provider in OmniRoute. Baidu Comate is Baidu's free AI coding assistant powered by their Wenxin (ERNIE) models, with **384,574 VS Marketplace installs** and **76,408 OpenVSX downloads** — major Chinese-market footprint.

### Why this passes the "native free tier only" filter

Per the extension docs, the **Personal Standard Edition (个人标准版) and Personal Professional Edition (个人专业版)** are free for individual developers via Baidu account signup. Enterprise tiers (企业标准版 / 企业专属版 / 企业私有化版) are paid and out of scope.

Free signup with Baidu account, no API key paste required.

### Extension details

- **VS Code Marketplace:** `BaiduComate.comate` ("文心快码 Baidu Comate") — 384,574 installs
- **Gitee variant:** `BaiduComate.comate-gitee` (9,007 installs) — for users behind Gitee infra
- **OpenVSX:** `BaiduComate.comate` (76,408 downloads)
- **Auth:** Baidu account login (free signup)
- **Backend:** Baidu's Wenxin (ERNIE) model family

### Distinctive feature: Zulu Agent

The standout feature is **Zulu (智能体)** — Baidu's coding agent that goes end-to-end from natural-language requirement to working code:

> "全自然语言实现需求，不写一行代码 (Realize requirements entirely in natural language, without writing a line of code)"  
> "理解代码库业务逻辑，帮助你迅速上手新项目 (Understands codebase business logic, helps you ramp on new projects quickly)"  
> "帮你自动搭建开发环境和启动服务 (Automatically sets up dev environments and starts services for you)"

That's more autonomous than most "agent mode" features in this batch — Zulu handles environment setup and service startup commands, not just code edits.

### Capabilities

1. **Zulu Agent** — end-to-end requirement-to-code generation with env setup
2. **Super Code Completion**:
   - Context-dependency-aware completion
   - Auto-correct typos and complete code
   - **Predict next cursor edit position** (forward-looking, unique)
   - Intelligent multi-line rewrite at predicted position
3. **Natural-language Codebase Chat**:
   - Whole-codebase business-logic analysis
   - Context-aware code generation
   - Enterprise knowledge-base integration (paid tiers)

### Languages supported (deep coverage)

Notable inclusions: **swan** (Baidu Smart Program), **San** (Baidu's framework), **VHDL** (hardware description), **mermaid** (diagram syntax) — these are unusual for general agents and well-served by Comate.

### Integration approach

1. **Auth extraction**: capture Baidu account session token after free signup
2. **MITM via Agent Bridge**: identify Comate's API endpoint
3. **Multi-capability routing**:
   - `comate/zulu-agent` for end-to-end task execution
   - `comate/completion` for super-completion with next-position prediction
   - `comate/chat` for codebase Q&A
4. **Next-position prediction**: surface as a distinct routing capability — most providers can't predict the *next edit location*, only complete at current cursor

### Privacy disclosure (from extension docs)

> "正常使用文心快码过程中需要利用正在编写的文件中的上文以及其他相关文件，来推理出接下来的下文。因此确实会需要通过网络将正在编写的代码文件内容传输给模型推理。我们承诺不使用这部分上传的代码做任何积累、分析、加工等后续动作。"

Translation: Code is uploaded to Baidu for inference but Baidu commits to not accumulating/analyzing/processing it for further use. Same trust model as most extension-scraper providers — flag in OmniRoute UI for transparency.
