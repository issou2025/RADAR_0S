# Project Proposal — Flutter App

## Project Understanding
You need assistance with: 
**macOS debug build shows restart screen when secure storage lacks Keychain entitlements**

*Project description:*
## Problem

When launching the current macOS debug build from `origin/main`, the app can show the startup failure/restart screen before the user reaches the app.

Observed during local launch:

```text
DatabaseEncryptionBootstrap.resolveCipherKey failed
PlatformException(Unexpected security result code, Code: -34018, Message: A required entitlement isn't present., -34018, null)
```

The failure happens while `flutter_secure_storage_macos` writes the database cipher key.

## Investigation Notes

The debug macOS build was ad-hoc/linker-signed without the expected entitlements. Inspecting the built app showed no useful team identifier/entitlement set. Local attempts to enable signing with Keychain Sharing were blocked by Apple provisioning/device registration for this machine, so the search investigation used a local-only debug bypass that was not committed.

This issue is separate from the NIP-50 search timeout fix in #5562.

## Expected Behavior

A developer should be able to build and launch the macOS debug app without hitting the restart screen due to missing Keychain entitlements.

## Acceptance Criteria

- macOS debug builds include the entitlements needed by secure storage/Keychain usage.
- The app can resolve or create the database cipher key on macOS debug launch without OSStatus `-34018`.
- The fix does not require committing developer-specific provisioning artifacts.
- Any native signing/config guardrails are updated if the accepted fix changes macOS entitlement handling.
