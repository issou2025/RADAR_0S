## Goal
Fix the iOS simulator build failure blocking local `flutter run` on this machine: an embedded binary's code signing doesn't match the parent app's, so debug builds can't launch on the simulator at all.

## Current state (verified on `feat/gamif-missions-ui`, 2026-06-28)
Running `flutter run -d <simulator> --dart-define=KOLABING_API_BASE_URL=http://127.0.0.1:8000/api/v1` builds successfully (`Xcode build done.`, ~165s) but then fails to launch:
```
Failed to build iOS app
Error (Xcode): Embedded binary is not signed with the same certificate as the parent app. Verify the embedded binary target's code sign settings match the parent app's.
Could not build the application for the simulator.
Error launching application on iPhone 16e.
```
This blocks any on-device/simulator verification of mobile changes on this machine, including the live repro needed for kolabing-app#48.

**Likely cause:** an embedded target (OneSignal notification-service extension, or a share extension, or similar) has a mismatched code-signing identity / provisioning profile relative to the `Runner` target — common after an Xcode or CocoaPods upgrade, or a switch between Development/Distribution signing on one target but not the other.

- [ ] Document the fix (which target, what setting) so it doesn't regress
