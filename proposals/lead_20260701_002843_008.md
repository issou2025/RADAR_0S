# Project Proposal — Structural Drawings

## Project Understanding
You need assistance with: 
**Feature Request / Proposal: Account Switcher Utility for Kiro IDE**

*Project description:*
Hi there,

I hope you are doing well.

I have been using Kiro IDE and found myself frequently needing to switch between multiple AWS SSO / Kiro accounts. Since Kiro is an Electron-based application, it locks session details inside the AppData\Roaming\Kiro cache and keeps the active token in C:\Users\<User>\.aws\sso\cache\kiro-auth-token.json. Switching accounts manually requires killing tasks, clearing specific directories, and swapping token files every single time.

To solve this, I've put together a prototype automation logic in Python using pywebview and shutil that successfully handles this process. It gracefully:

Closes Kiro IDE and all its sub-processes completely (taskkill /F /T).

Clears session/cookie directories (like Local Storage, Session Storage, Cache, Network, etc.) to prevent account conflicts.

Swaps/backs up the kiro-auth-token.json file.

Restarts the IDE either with a selected account or in a clean "Guest/Add Account" mode.

I was wondering if you would be interested in creating a dedicated, polished utility tool or a desktop app for this purpose, or perhaps integrating such a mechanism if you have a relevant repository? I can share the current Python script logic, HTML/CSS UI structure, and folder targets with you to speed up the process.

Looking forward to hearing your thoughts!
