### What happened?

**Expected**
A transient GPS spike should not cause a geofence EXIT if the device has not actually left the property.

**Actual / concern**
Geofence monitoring evaluates inside/outside based on reported coordinates. A single bad fix that places the device outside the radius can fire an EXIT event — this applies to:
- start() (continuous tracking + geofences)
- startGeofences() (geofence-only mode)

**Our app-level mitigation (not ideal)**
We currently guard EXIT in Dart:
1. Reject EXIT if distance - accuracy <= geofenceRadius (device might still be inside given GPS error)
2. Request a fresh high-accuracy fix via Geolocator.getCurrentPosition() and re-check before finishing the job

This catches many cases, including “confidently wrong” spikes (e.g. ±20 m accuracy but 400 m off). But:
- Every consumer must implement this themselves
- Headless/terminated scenarios are harder to validate consistently
- A spike with falsely good accuracy that coincides with a bad confirmation fix could still slip through

**Desired SDK behavior**
Built-in spike filtering before geofence EXIT is emitted, e.g.:
- Reject fixes with implausible implied speed from last good fix
- Require N consecutive outside fixes before EXIT
- Minimum dwell time outside fence before EXIT
- Optional accuracy ceiling for geofence transitions

Real-user GPS spike while stationary on site. Reported position jumped far outside the property; with smaller geofence (which is usually 75m) this would emit an EXIT event at the SDK level.

        debug: false,
        geofenceModeHighAccuracy: true,
        geofenceInitialTriggerEntry: true,
        showsBackgroundLocationIndicator: true,
        startOnBoot: true,
        stopOnTerminate: false,
        enableHeadless: true,

        },
        authorization: await authorization,
        httpRootProperty: '.',
        autoSync: true,
        autoSyncThreshold: 0,
        geofenceTemplate:

        identifier: 'property_geofence',

        longitude: longitude,
        radius: geolocationPerimeter,

        notifyOnDwell: false,
        notifyOnEntry: false,

          'job_id': jobId,
