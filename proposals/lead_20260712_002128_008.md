# Summary

The Riverpod 2.6.1 → 3.3.2 migration tracked in #462 is **blocked by the Dart SDK version**. This issue tracks the prerequisite work: upgrading the Flutter SDK from 3.35.7 (Dart 3.9.2) to Flutter 3.36+ (Dart 3.10+).

Unblocks: #462

---

## Why this is needed

During the second Riverpod migration attempt (documented in [#462 comment](https://github.com/NassarAlshabi1/marina-hotel-wit-app/issues/462#issuecomment-4948375005)), `flutter pub get` failed with:

```
Because test >=1.31.1 requires SDK version >=3.10.0 <4.0.0 ...
And because flutter_riverpod >=3.3.2 depends on riverpod 3.3.2
which depends on test ^1.0.0, ...
version solving failed.
```

The root cause: `riverpod 3.3.2` declares `test ^1.0.0` as a regular dependency (likely a packaging mistake — should be `dev_dependencies`), and `test >=1.31.1` requires Dart SDK 3.10+. The project currently uses Dart 3.9.2 (Flutter 3.35.7).

Even if the 10 `StateNotifierProvider` rewrites were completed, Riverpod 3.3.2 still cannot be installed without first upgrading the Dart SDK.

## Required changes

### 1. Update `mobile/pubspec.yaml`

```yaml
environment:
  sdk: '>=3.10.0 <4.0.0'    # was: '>=3.8.0 <4.0.0'
  flutter: '>=3.36.0'        # was: '>=3.35.7'
```

### 2. Update CI workflows

| Workflow | Current | Target |
|---|---|---|
| `.github/workflows/main.yml` | `flutter-version: 3.41.0` (already newer!) | `3.41.0` or latest stable |
| `.github/workflows/ci.yml` | (uses `subosito/flutter-action@v2` channel: stable) | Pin to specific version for reproducibility |
| `.github/workflows/security-scan.yml` | (uses `subosito/flutter-action@v2` channel: stable) | Same |

Note: `main.yml` already uses Flutter 3.41.0 (Dart 3.12+) in CI, so CI may already work. The `pubspec.yaml` SDK constraint `>=3.8.0` is the actual blocker — it allows the local environment to use older Flutter versions that can't resolve Riverpod 3.3.2's deps.

### 3. Verify all dependencies on Dart 3.10+

Some packages may have constraints that need updating. Run:
```bash
cd mobile
flutter pub outdated
flutter pub upgrade --major-versions
flutter analyze
flutter test
```

### 4. Local development environment

Update the Flutter SDK on all developer machines:
```bash
flutter upgrade
flutter --version  # must show Flutter 3.36+ / Dart 3.10+
```
